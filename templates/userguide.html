📝 User Guide: Autoflow Localization Web App

URL: https://autoflow-8667.onrender.com

🚀 What This App Does

This web app automates localization workflows including:

  • 🛠 Generating bilingual XLIFFs from JSON/Properties
  • 📥 Converting translated XLIFFs back into localized files
  • 📊 Comparing final source vs. target files before delivery
  • 📦 Supports multi-language batch processing

🧭 Step-by-Step

  1. Select Workflow – TEP or Legacy
  2. Select Process Type – Preprocess or Postprocess
  3. Upload Files – based on the selected mode
  4. Click Start – wait for processing
  5. Download Results – file links + ZIP download

🔁 Workflow Details

✅ TEP – Preprocess

  • Upload .json or .properties files
  • Output: bilingual XLIFF files for translation

✅ TEP – Postprocess

  • Upload translated .xliff files
  • Output: localized JSON/Properties files renamed with language (e.g. MyFile-Tamil.json)

🟠 Legacy – Preprocess

  • Upload source files (.json or .properties)
  • Upload ZIP of target files organized like:
    hi-IN/file.json, ta-IN/file.properties
  • Output: bilingual XLIFF files (only for matching keys)

🟠 Legacy – Postprocess

  • Upload translated bilingual .xliff files
  • Output: updated localized files named like MyFile-Hindi.json

🔍 Final Compare

  • Upload source files (.json/.properties)
  • Upload ZIP of translated files (folder-wise: ta-IN/, hi-IN/)
  • Output: Excel report + preview table

📦 Output

  • Files are listed with individual download links
  • Full ZIP available as batch.zip

⚠️ Common Errors

  • Missing Source or Target: Required files not uploaded
  • ZIP Error: ZIP folder structure not valid (use language-code folders)
  • XLIFF Mismatch: Keys not matched between XLIFF and original

✅ Notes

  • TEP: Use for new content translation workflows
  • Legacy: Use when target already exists and needs review
  • ZIP: Only required for Legacy Preprocess and Final Compare
  • Languages: Detected automatically from XLIFF metadata
